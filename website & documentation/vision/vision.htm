1          Introduction

1.1           Purpose

The purpose of this document is to define the high-level requirements of the Resume Builder in terms of the needs of the end users.

1.2           Scope

This Vision Document applies to the Resume Builder, which will be developed by our team. It will be built on python and it's GUI architecture.

Resume Builder will enable students to generate resume based on the information they provide in the application in word,pdf and web format

2          Positioning

2.1           Business Opportunity

This project will be presenting a new standalone PC application for students to build their resume and simple web protfolio.

The current resume builders for PC are either paid or available online. Our application will enable student to create their resume on any PC.

Our application is based on the principle of FOSS i.e., Free Open Source Software.

5          Product Features

This section defines and describes the features of the Resume Builder. Features are the high-level capabilities of the system that are necessary to deliver benefits to the users.

5.1           Fill Info

The user must be able to select the info he need on his resume and fill relevant information.

5.2           Generate Word Format Resume.

The user should be able to generate his resume in word format after he has filled out the relevant information without any error.

5.3           Generate PDF

The user should be able to generate PDF file for his resume provided he has filled ut the relevant information.

5.4           Generate Web Page

The user should be able to generate web page for his resume.

5.5           Generate All Format

TThe user should be able to generate word, pdf and web page for his resume.

5.6           View Help

The user should be able to view help content provided with the application.

5.7           Report Error and feedback

The user should be able to report any errors encountered or provide feedback.

7          Quality Ranges

This section defines the quality ranges for performance, robustness, fault tolerance, usability, and similar characteristics for the Resume Builder.

Availability: The System is meant to run on user's PC.

Usability: The System shall be easy-to-use and shall be appropriate for the target market of students and other users who want to build resume.

Usability: The System shall include help for the user. SError reporting should also be included.

Maintainability: The System shall be designed for ease of maintenance.

8          Precedence and Priority

This section provides some direction on the relative importance of the proposed system features. The features defined in this Vision Document should be included in the first 2 releases of the system. All features critical to basic resume building are planned for the first release.

As development progresses on this system, the feature attributes (referenced in Section 7 of this document) will be used to weight the relative importance of the features and to plan the release content. The benefit, effort, and risk attributes are used to determine priority of a feature and target release.

Release 1 must contain as a minimum the basic functionality as listed below:

    • Enter Info
    • Generate Word format.
    • Generate PDF format
    • Generate Web page.

Release 2 should include:

    • Ability to fetch existing resume.
    • Edit resume built using this application
    • Custom styling the WEB portfolio.

The functionality for Release 3 has not yet been determined. It is anticipated that this release will contain enhancements to the existing functionality.
